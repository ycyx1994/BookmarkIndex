## 新增需求

### 需求:系统必须支持通过设置面板导入 HTML 书签文件
系统必须在设置面板中提供导入功能，支持 Netscape Bookmark 格式的 HTML 文件导入。

#### 场景:打开设置面板
- **当** 用户点击侧边栏底部的"设置"按钮
- **那么** 系统必须显示设置面板（弹窗或侧滑面板）
- **并且** 面板中包含"导入书签"选项

#### 场景:选择 HTML 文件
- **当** 用户在设置面板中点击"选择文件"并选择一个 HTML 文件
- **那么** 系统必须解析该 HTML 文件中的书签数据
- **并且** 提取所有书签（标题、URL）和文件夹结构

#### 场景:显示导入确认对话框
- **当** 系统解析 HTML 文件完成时
- **那么** 系统必须显示二次确认对话框
- **并且** 对话框中显示将导入的书签数量和分类数量
- **并且** 提示"这将替换当前所有导入的书签数据，Chrome 原生书签不受影响"
- **并且** 提供"导出当前数据作为备份"的勾选框（默认勾选）

#### 场景:确认导入
- **当** 用户点击"确认导入"
- **那么** 系统必须将解析后的书签数据写入 chrome.storage.local 的 `importedBookmarks` 和 `importedFolders` key
- **并且** 替换之前所有导入的数据
- **并且** 重新合并 Chrome 书签和导入书签，刷新页面显示

#### 场景:取消导入
- **当** 用户点击"取消"
- **那么** 系统必须关闭确认对话框，不执行任何数据变更

### 需求:系统必须支持导出当前书签数据为 HTML 文件
系统必须在设置面板中提供导出功能，将当前书签数据导出为 Netscape Bookmark 格式的 HTML 文件。

#### 场景:点击导出按钮
- **当** 用户在设置面板中点击"导出书签"
- **那么** 系统必须将所有书签（Chrome + 导入）转换为 Netscape Bookmark HTML 格式
- **并且** 触发浏览器下载，文件名为 `bookmarks-export-YYYY-MM-DD.html`

### 需求:系统必须支持清除推荐数据
系统必须在设置面板中提供清除点击频次统计的功能。

#### 场景:清除推荐数据
- **当** 用户在设置面板中点击"清除推荐数据"
- **那么** 系统必须清空 chrome.storage.local 中的 `clickStats` 数据
- **并且** 推荐区块不再显示
