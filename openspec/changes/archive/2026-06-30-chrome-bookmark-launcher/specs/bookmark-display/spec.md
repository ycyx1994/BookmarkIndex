## 新增需求

### 需求:新标签页必须显示书签启动器界面
系统必须通过 `chrome_url_overrides.newtab` 替换 Chrome 默认新标签页，显示书签启动器。

#### 场景:用户打开新标签页
- **当** 用户在 Chrome 中打开新标签页
- **那么** 系统必须显示书签启动器界面
- **并且** 包含顶部搜索栏、左侧分类侧边栏、右侧书签卡片网格

### 需求:左侧侧边栏必须显示分类列表
系统必须在左侧显示所有书签分类（文件夹），每个分类显示名称和书签数量。

#### 场景:显示分类列表
- **当** 书签数据加载完成时
- **那么** 侧边栏必须显示"全部书签"选项（带总数量）
- **并且** 显示所有顶层文件夹名称及各自书签数量

#### 场景:默认选中"全部书签"
- **当** 用户首次打开新标签页时
- **那么** "全部书签"必须为默认选中状态
- **并且** 右侧显示所有分类的书签

#### 场景:设置按钮位于侧边栏底部
- **当** 侧边栏渲染完成时
- **那么** 侧边栏底部必须显示"设置"按钮

### 需求:点击侧边栏分类必须过滤右侧书签
系统必须采用过滤模式：点击分类后右侧只显示该分类下的书签。

#### 场景:点击某个分类
- **当** 用户点击侧边栏中的某个分类（如"工作"）
- **那么** 该分类必须高亮显示
- **并且** 右侧必须只显示该分类下的书签卡片
- **并且** 右侧顶部必须显示分类标题和书签数量

#### 场景:点击"全部书签"
- **当** 用户点击侧边栏中的"全部书签"
- **那么** 右侧必须显示所有分类的书签
- **并且** 按分类分组展示，每组显示分类标题

### 需求:右侧必须以卡片网格展示书签
系统必须在右侧主区域以网格布局展示书签卡片，每张卡片包含 favicon 图标、标题和域名。

#### 场景:渲染书签卡片
- **当** 书签数据加载完成时
- **那么** 每张卡片必须显示网站 favicon（通过 Google Favicon API 获取）
- **并且** 显示书签标题（单行，溢出省略）
- **并且** 显示域名（如 github.com）

#### 场景:点击卡片打开链接
- **当** 用户点击某张书签卡片
- **那么** 系统必须在新标签页中打开对应书签 URL

#### 场景:Favicon 加载失败
- **当** Google Favicon API 无法返回某网站的图标时
- **那么** 系统必须隐藏该 favicon 图片（不显示破损图标）

### 需求:UI 必须采用深色主题
系统必须使用深色主题配色，参考 Markoob 书签启动器风格。

#### 场景:深色主题配色
- **当** 页面渲染时
- **那么** 背景色必须为深蓝色调（#1a1a2e）
- **并且** 侧边栏背景为 #16213e
- **并且** 卡片背景为 #0f3460，hover 时为 #1a4a7a
- **并且** 强调色为 #4a9eff
- **并且** 主文字色为 #e8e8e8，副文字色为 #a0a0b0

#### 场景:卡片 hover 效果
- **当** 用户鼠标悬停在书签卡片上时
- **那么** 卡片必须有轻微上移和阴影增强效果
