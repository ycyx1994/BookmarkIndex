## 新增需求

### 需求:系统必须基于点击频次推荐书签
系统必须记录用户点击书签卡片的频次，展示 Top N 高频书签作为推荐。

#### 场景:记录点击事件
- **当** 用户点击任意书签卡片
- **那么** 系统必须将该书签 URL 的点击次数 +1
- **并且** 存储到 chrome.storage.local 的 `clickStats` key 中

#### 场景:显示推荐书签
- **当** 有点击记录且不在搜索状态时
- **那么** 右侧主区域顶部必须显示"推荐书签"区块
- **并且** 按点击频次降序排列，最多显示 20 个
- **并且** 每个推荐卡片显示 favicon、标题、域名

#### 场景:无点击记录时隐藏推荐
- **当** 用户从未点击过任何书签时
- **那么** 推荐区块必须隐藏，不显示空白区域

#### 场景:搜索时隐藏推荐
- **当** 用户在搜索状态时
- **那么** 推荐区块必须隐藏，只显示搜索结果
