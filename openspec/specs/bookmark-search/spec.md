## 新增需求

### 需求:顶部搜索栏必须支持跨分类搜索书签
系统必须在页面顶部显示搜索栏，支持按书签标题和 URL 进行关键词搜索，搜索范围为所有分类。

#### 场景:输入关键词搜索
- **当** 用户在搜索栏输入关键词
- **那么** 系统必须实时过滤并显示标题或 URL 包含关键词的书签
- **并且** 搜索结果忽略当前选中的分类，覆盖所有分类
- **并且** 侧边栏不高亮任何分类

#### 场景:搜索结果为空
- **当** 用户输入的关键词没有匹配任何书签
- **那么** 系统必须显示"未找到匹配的书签"提示
- **并且** 右侧不显示任何卡片

#### 场景:清空搜索恢复视图
- **当** 用户清空搜索栏内容
- **那么** 系统必须恢复到搜索前的状态（显示当前选中分类的书签或全部书签）

### 需求:搜索必须支持防抖
系统必须对搜索输入进行防抖处理，避免每次按键都触发过滤。

#### 场景:快速输入时防抖
- **当** 用户快速连续输入多个字符
- **那么** 系统必须等待用户停止输入 200ms 后才执行搜索
- **并且** 不会在中间状态触发过滤

### 需求:搜索栏必须支持快捷操作
搜索栏必须支持清除按钮和 Escape 键快捷操作。

#### 场景:显示清除按钮
- **当** 搜索栏有输入内容时
- **那么** 搜索栏右侧必须显示清除按钮（×）

#### 场景:点击清除按钮
- **当** 用户点击清除按钮
- **那么** 搜索栏内容必须被清空
- **并且** 恢复到搜索前的视图状态

#### 场景:按 Escape 键
- **当** 用户在搜索栏聚焦状态下按 Escape 键
- **那么** 搜索栏内容必须被清空
- **并且** 搜索栏失去焦点
